## Learnings / Inefficiencies

**2026-07-03 — ads.sh audit defaults to the MCC and fails**
- Observation: `ads.sh audit` with no `--customer` targets the manager account → `REQUESTED_METRICS_FOR_MANAGER` ("Metrics cannot be requested for a manager account"). Must pass the leaf (Baymo `8911925499`). Also: `apply-fixes`/`update` are dry-run by default; live mutation needs `--apply`.
- Action: Always pass the leaf `--customer`; use `--apply` to mutate.
- Confidence: high

**2026-07-03 — Firestore prod 403 on signup = rules never deployed to prod**
- Observation: `.firebaserc` default project is `vonteva-dev`; any `firebase deploy` without `--project prod` leaves `vonteva-prod` on the locked default ruleset. Worse, `early-access.ts:178` maps `permission-denied` to `{kind:'duplicate'}`, so the modal silently advances and dropped signups look successful.
- Action: `firebase deploy --only firestore:rules --project prod` (note `--only firestore:rules`, not `firestore:rules` bare); verify a real doc lands in the console, don't trust UI advance.
- Confidence: high

**2026-07-03 — GrowthBook shows zero users when attribution uses user_id on anonymous traffic**
- Observation: Landing pages are anonymous; the SDK `autoAttributesPlugin` only sets `gbuuid`/device_id, never `user_id`, so experiments keyed on user_id show no users. `assignmentQueryId` is a required config field (optional produced opaque 400s); `str(HTTPStatusError)` had hidden GrowthBook's real error body.
- Action: Set `assignmentQueryId` to a device/anonymous query in each `experiments/*.yaml` + `gb.config.json`; keep HTTP-error-body surfacing.
- Confidence: high

**2026-07-03 — Run gb/ads Python only through gb.sh / correct venv**
- Observation: Raw `uv` created stray `.venv` dirs indexed as phantom skills, and `uv sync` stripped pytest. The ads venv is `.venv-ads` (not `.venv`); wrong interpreter → `ModuleNotFoundError: yaml/google`. pyright "unresolved import" warnings are false positives from the wrong interpreter.
- Action: Invoke via `gb.sh` / `ads.sh`; for ads tests, `uv run --project . --extra dev` with `UV_PROJECT_ENVIRONMENT=.venv-ads`.
- Confidence: high

**2026-07-03 — whois gives false "available" on Identity Digital registries**
- Observation: `.build .guru .live` etc. prepend the registry's own record, so naive parsing reads "available." Real signal is the per-domain `DOMAIN NOT FOUND` string vs a known-free control. RDAP-via-proxy unreliable (404 ≠ available).
- Action: Use `DOMAIN NOT FOUND` control-string comparison, not a parser.
- Confidence: high

**2026-07-03 — tailwind prod build ships unminified via `npx vite build`**
- Observation: `npm run build` runs `npx vite build`, which leaves `NODE_ENV` undefined at config-factory time, so cssnano never runs. Fixed by `tailwindPlugin(mode === 'production')` instead of relying on Vite's env side effect.
- Confidence: medium
