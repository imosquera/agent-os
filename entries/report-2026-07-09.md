## Learnings / Inefficiencies

**2026-07-09 — adkit `ads_skill/lib/` is git-ignored in the source repo (imosquera/adkit) → every fresh clone/sync is broken**
- Observation: `.gitignore:17` has a blanket `lib/` rule from the standard Python template (meant for venv `lib64/`), which shadows the real `skills/adkit/scripts/ads_skill/lib/` subpackage (`auth.py`, `cluster.py`, `report.py`, …). Confirmed via `git check-ignore -v` → `.gitignore:17:lib/`. The dir was never committed, so `bin/preflight.py`/`audit.py` fail with `"ads_skill package not importable"` for anyone cloning fresh or via `npx skills add`. - Action: Fix in **imosquera/adkit**: anchor the rule (`/lib/` or `.venv/lib/`), `git add -f skills/adkit/scripts/ads_skill/lib/`, verify with a clean clone, then re-sync consumers. Audit `.gitignore` for other blanket dir names (`build/`, `dist/`, `share/`) shadowing real source.
- Confidence: high

**2026-07-09 — adkit `keyword-ideas` returns zero for all seeds and silently emits `[]` (blocks `/adkit gtm`)**
- Observation: Post-TS-refactor, `ads.sh keyword-ideas --seed "salon booking app" --seed "appointment reminder"` (customer 8911925499) hits `GenerateKeywordIdeas`, gets **zero rows with no exception**, prints `API returned zero ideas; using LLM seeds only`, then writes `[]` — so gtm has nothing to tier. The "LLM seeds only" fallback is a no-op (`buildCandidateDicts` drops seeds with no API metric). Broad seeds that worked before the rewrite (proof: tonewell processed file has real KP-decorated data) now return zero, so the refactored `buildRequest` request path is suspect.
- Action: Log the actual `IGenerateKeywordIdeasRequest` + raw response; check `keyword_plan_network`, geo/language constants, `keyword_seed` shape. Make the fallback emit undecorated seeds or exit non-zero with the request echoed — never a silent empty `[]`.
- Confidence: high

**2026-07-09 — adkit `audit.ts` type-lie: ~7 unguarded nested Google-Ads field accesses throw on real campaign data**
- Observation: `src/bin/audit.ts` type declarations (L108–158) mark optional API sub-objects as required, and code accesses them unguarded: `quality_info.quality_score` (747), `policy_summary.policy_topic_entries` (872), `metrics.average_cpc` (625), search-term `metrics.*` (686), `landingPageMobile` metrics (840/844), `search_impression_share` (521/541/575), RSA `headlines` (453). Each throws when the field is absent (keyword with no QS/spend, non-RSA ad, campaign row missing metrics) — which is why the committed `audit.js` crashed on this campaign with `quality_score`.
- Action: Guard each nested access and make the type decls honest (optional). Fix belongs in **imosquera/adkit** `src/bin/audit.ts`; L747/L872 are already patched in the local vendored copy but not upstream.
- Confidence: high

**2026-07-09 — adkit run-as-main guard silently no-ops through the `.claude`→`.agents` symlink (reconfirmed)**
- Observation: Every `src/bin/*.ts` ends with `if (import.meta.url === \`file://${process.argv[1]}\`)`. Because `.claude/skills/adkit` is a symlink to `.agents/skills/adkit`, the launch path ≠ Node's realpath, the guard never matches, `main()` never runs, exit 0 with no output. `ads.sh` set `SCRIPT_DIR` via `pwd` (logical) → same mismatch; worked around with `pwd` → `pwd -P` (uncommitted). Also `tsup.config.ts:17` `format:["esm","cjs"]` makes the `.cjs` bins no-op the same way.
- Action: Already in memory ([[adkit_symlink_runguard_noop]]). Proper fix: compare `fs.realpathSync` of both sides in every bin; add a test invoking a bin through a symlinked path asserting output.
- Confidence: high

**2026-07-09 — Tailwind Play CDN on the 26 idea pages is a Core Web Vitals / mobile-friendly drag**
- Observation: `cdn.tailwindcss.com` ships the full JIT compiler and recompiles the class set in-browser on every load — a render-blocking script Tailwind's own docs flag as prototyping-only. It delays LCP and can flash unstyled content, feeding the same `mobile_unfriendly_clicks` signal the audit reads. Google Fonts are already loaded correctly (preconnect + display=swap), so they're not the culprit.
- Action: Precompile Tailwind via the **existing** `vite build` (no new tooling) to a static `dist/tailwind.css`, replace the `<script>` CDN tag with a `<link>` across all 26 `src/ideas/*.html`. Repo-wide fix, not per-campaign; scope as a speckit spec.
- Confidence: medium

## Patterns & Anti-patterns

- **Silent-empty / silent-no-op is adkit's dominant failure mode.** The symlink
  run-guard (exit 0, no output), `keyword-ideas` emitting `[]`, and the "LLM
  seeds only" no-op fallback all *look like success and produce nothing*.
  Convention: any adkit path that yields zero results must either echo its
  request and exit non-zero, or emit an explicit diagnostic — never a bare empty
  success. This is the same class as the earlier Firestore `permission-denied` →
  silent-`duplicate` collapse.
- **Type declarations that lie about API optionality are latent crashes.** The
  `audit.ts` L108–158 required-field decls over genuinely-optional Google Ads
  fields is a parse-don't-validate violation: trust-boundary data (the API
  response) was typed as if already validated. Convention: model external API
  responses with optional fields and guard at access, per the constitution's
  Principle I.
